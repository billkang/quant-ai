# 数据管理

数据管理模块整合了「数据源」、「渠道管理」、「事件查询」、「采集监控」和「板块管理」五个子功能，提供统一的数据资产管理界面。

## 数据源

「数据源」页面展示系统内置的六大基础数据源，每个数据源下包含多个采集渠道：

- **A股个股信息**：东方财富个股新闻、个股公告、A股实时行情、个股财务数据
- **港股信息**：港股新闻、港股行情
- **国际新闻**：国际市场指数
- **财经资讯**：新浪财经 RSS、华尔街见闻 API
- **板块数据**：东方财富板块资金流
- **宏观数据**：中国宏观经济指标（CPI、PPI、PMI 等）

数据源为「容器」概念，定义调度策略和分类。点击每行可展开，通过**多选下拉框**选择该数据源要使用的采集渠道。同一个渠道可以被多个数据源复用。

> **提示**：如果数据源未选择任何渠道，触发采集时将自动回退到该数据源下的默认渠道。

## 渠道管理

「渠道管理」页面用于查看和管理所有采集渠道。每个渠道默认归属一个数据源，但可以通过数据源页面被多个数据源引用：

### 采集方式

| 方式 | 说明 | 示例 |
|------|------|------|
| API | 调用第三方数据接口 | Yahoo Finance、华尔街见闻 |
| RSS | 订阅 RSS 源 | 新浪财经热榜 |
| 爬虫 | 网页抓取 | 自定义站点采集 |
| AkShare | 调用 AkShare 数据接口 | 东方财富新闻、宏观数据 |

### 渠道字段

- **所属数据源**：渠道归属的父级数据源
- **名称**：渠道显示名称
- **采集方式**：api / rss / crawler / akshare
- **端点地址**：接口地址或 RSS URL
- **超时**：请求超时时间（秒）
- **配置 JSON**：额外参数，如 `stock_pool`、`max_pages`、`fields` 等
- **启用状态**：停用后调度器不会自动采集该渠道

### 渠道配置示例

```json
{
  "source": "eastmoney",
  "stock_pool": "watchlist+backtest",
  "max_pages": 3
}
```

## 事件查询

「事件查询」页面用于浏览和管理系统采集到的事件信号，包括宏观事件、行业事件和公司事件。

### 事件筛选

- 按股票代码搜索
- 按事件范围筛选（个股 / 板块 / 市场）
- 按数据源筛选
- 按时间范围筛选

### 事件详情

点击事件标题可查看详情，包括情感得分、强度、确定性、紧急度、标签和原始信号。

## 采集监控

「采集监控」页面提供采集任务的实时监控与历史查询，采用三级钻取结构：

### 列表视图

按渠道聚合展示统计数据：
- 渠道名称
- 所属数据源
- 总任务数 / 成功数 / 失败数
- 最近采集时间

点击「详情」进入该渠道的任务列表。

### 任务列表

展示某个渠道的所有采集任务：
- 任务状态（成功 / 失败 / 运行中）
- 触发方式（自动 / 人工）
- 新事件数、重复数、错误数
- 开始时间

点击「查看内容」进入任务详情。

### 任务详情

展示单个任务的完整信息：
- 任务状态与日志
- 采集到的事件列表（标题、股票、情感得分、时间）

### 监控筛选

- **时间范围**：选择采集任务的起止日期
- **采集分类**：筛选自动采集或人工触发的任务

## 板块管理

「板块管理」页面展示 CSRC 行业分类一级板块列表，支持启用/停用管理：

- 仅启用的板块会参与板块轮动数据采集
- 系统默认启用全部 CSRC 一级行业
- 可通过开关灵活控制关注板块

## 使用技巧

- 停用不常用的采集渠道可减少系统资源占用和外部 API 调用
- 板块管理中只启用关注的板块，可提升板块数据采集效率
- 大规模手动采集建议在非交易时段执行
- 定期查看采集监控的失败记录，及时调整渠道配置或检查外部接口状态
- 结合「规则管理」设置事件触发的自动告警
