# 数据管理

数据管理模块用于监控系统数据质量并执行必要的维护操作。

## 数据概览

展示当前数据资产的整体情况：
- **股票价格**：覆盖的股票数量与最新更新日期
- **技术指标**：已计算的指标记录数
- **基本面数据**：财报与财务指标覆盖情况
- **事件数据**：已采集的事件数量

## 数据补录

当发现数据缺失时，可手动触发补录：
1. 选择数据类型（价格/指标/事件）
2. 输入股票代码与时间范围
3. 点击 **补录数据**

## 数据质量检查

系统自动执行的检查项：
- 价格连续性：检测停牌、缺失交易日
- 指标一致性：验证技术指标计算结果是否合理
- 基本面及时性：检查财报数据是否按期更新

检查结果在页面上以色块标识：
- 绿色：正常
- 黄色：警告（数据延迟）
- 红色：异常（数据缺失或错误）

## 采集任务监控

在 **采集任务** 页面查看所有定时任务的运行状态：
- 任务名称与描述
- 执行周期（每小时/每日/每周）
- 上次执行结果（成功/失败）
- 下次执行时间

支持手动触发任务执行。

## 数据源配置

在 **数据源配置** 页面管理外部数据接口：
- 东方财富行情接口
- Yahoo Finance 接口
- 财经新闻 RSS 源
- 宏观经济数据 API

可启用/禁用特定来源，调整请求频率限制。

## 使用技巧

- 数据质量异常时优先检查数据源配置中的接口状态
- 大规模补录建议在非交易时段执行，避免占用资源
- 定期查看采集任务日志，及时发现接口变更导致的失败
