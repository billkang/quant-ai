# Proposal: 量化分析引擎 (Quantitative Analysis Engine)

## 背景

当前系统是一个"AI 选股助手"——具备行情查看、新闻聚合、AI 文本诊断能力，但缺乏量化分析的核心要素：**可重复、可验证、基于数据的规则与计算**。LLM 可以给出"趋势上行"的判断，但这基于文本生成而非真实指标计算。

目标是将系统升级为**私人量化分析平台**：数据驱动、策略可回测、决策可追溯。

## 目标

构建一套完整的量化分析基础设施，覆盖从数据获取、指标计算、策略回测到组合分析的完整链路。

## 非目标

- **不做交易执行**：本系统只做分析和建议，不连接券商 API
- **不做高频/实时策略**：面向中低频个人投资者（日线/周线级别）
- **不做多因子模型**：首期聚焦经典技术分析 + 基本面筛选

## 范围

### In Scope

1. 历史行情数据持久化（日K + 指标）
2. 技术指标计算引擎（MA/RSI/MACD/KDJ/布林带/成交量）
3. 基本面数据获取与存储（PE/PB/ROE/营收增速等）
4. 策略回测框架（单策略、经典信号、绩效报告）
5. 组合风险分析（夏普比率、最大回撤、相关性矩阵）
6. 告警系统（价格突破、指标信号、新闻情绪）
7. 数据 Pipeline 自动化（收盘后自动更新）
8. 停牌检测与前复权处理
9. 可视化增强（指标叠加、收益曲线、热力图）
10. AI 诊断增强（结合真实指标值而非纯文本生成）

### Out of Scope

- 交易执行/模拟盘对接
- 机器学习预测模型
- 多因子选股模型
- 实时 Level-2 行情

## 影响

- 新增 3-4 张数据库表
- 新增核心服务模块：indicator_service, backtest_service, fundamental_service
- scheduler 从"placeholder"变为真正的数据 Pipeline
- 前端新增：回测页面、策略管理、组合风险分析、指标详情页
